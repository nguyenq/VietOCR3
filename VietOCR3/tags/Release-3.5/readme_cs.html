VietOCR

POPIS

VietOCR je rozhraní Java pro systém rozpoznávání znaků Tesseract, poskytující podporu rozpoznávání znaků pro běžné formáty obrázků a vícestranové obrázky. Program obsahuje funkci dalšího zpracování, která pomáhá při opravování chyb, které se pravidelně objavují při procesu rozpoznávání, a tak se zvyšuje míra přesnosti výsledku. Program je též možné použít jako konzolovou aplikaci, kterou je možné spustit z příkazového řádku.

Podporováno je nyní i dávkové zpracování. Program sleduje nové obrázkové soubory ve sledované složce a automaticky je zpracuje pomocí OCR systému a výstup uloží do výstupní složky.

SYSTÉMOVÉ POŽADAVKY

Java Runtime Environment 6.0 anebo vyšší verze.

INSTALACE

Windows verze Tesseract 3.02 je součástí programu. Další jazykové datové balíky pro Tesseract, které začínají kódem ISO639-3, by měly být umístěny do podadresáře tessdata.

V Ubuntu jsou Tesseract a jeho jazyková data součástí Graphics (universe) repository. Můžete je nainstalovat pomocí Synaptic anebo příkazu:

sudo apt-get install tesseract-ocr tesseract-ocr-eng tesseract-ocr-vie

Soubory budou umístěny do /usr/bin resp. /usr/share/tesseract-ocr/tessdata. Pokud si Tesseract sestavíte sami ze zdrojových kódov, nainstaluje se do /usr/local/bin a /usr/local/share/tessdata. Z tohoto důvodu může být potřebné nastavit umístění programu Tesseract v Nastavení VietOCR. Pokud jsou data umístěna nestandardně, je potřeba je nalinkovat do tessdata. Též je možné použít proměnnou prostředí TESSDATA_PREFIX na informování VietOCR, kde jsou umístěna data:export TESSDATA_PREFIX=/usr/local/share/Informace k instalaci programu Tesseract na jiných platformách hledejte na stránce wiki Tesseract.

VietOCR taktéž poskytuje podporu pro stahování a instalaci vybraných jazykových balíčků přes položku nabídky Stáhnout jazyková data. V závislosti na umístění složky tessdata, může být potřeba spustit program jako superuživatel anebo administrátor, aby bylo možné nainstalovat stáhnutá data do složky, pokud je v systému složek, jako například v /usr na Linuxe anebo C:\Program Files v systému Windows.

Podpora snímání je poskytována přes knihovnu Windows Image Acquisition Library v2.0, která vyžaduje Windows XP Service Pack 1 (SP1) anebo vyšší verzi. Tato knihovna je standardní součástí Windows Vista a 7. Instalaci na Windows XP udělejte následovnými kroky: zkopírujte soubor wiaaut.dll do složky System32 (obyčejně je to C:\Windows\System32) a spusťte následující příkaz v příkazovém řádku:

regsvr32 C:\Windows\System32\wiaaut.dll

V systému Linux vyžaduje snímání instalaci balíčků SANE:

sudo apt-get install libsane sane sane-utils libsane-extras xsane

POKYNY

Program spustíte příkazem:

java -jar VietOCR.jar

Poznámka:

Pokud narazíte na problém „out-of-memory exception“, spusťte skript ocr(.bat) namísto uvedeného příkazu.Obrázky určené pro rozpoznávání by měly být snímány v rozlišení aspoň 200 DPI (bodů na palec - dots per inch) až 400 DPI v monochromatickém (černobílém) režimu anebo v odstínech šedé. Snímání ve vyšším rozlišení nepřináší při rozpoznávání znaků vyšší přesnost. Míra přesnosti však závisí hlavně na kvalitě nasnímaného obrázku. Typické nastavení pro snímání je 300 DPI a 1 bpp (bit per pixel) černo-bílo anebo 8 bpp (odstíny šedé) do nekomprimovaného TIFF anebo PNG formátu.Nový Režim snímku obrazovky nabízí lepší míru přesnosti pro obrázky s nízkým rozlišením, jako např. snímky obrazovky, pomocí zvětšení rozlišení na 300 DPI.

Vedle vestavěného algoritmu pro další zpracování textu si můžete přidat vlastní schéma nahrazování textu - textový soubor s názvem x.DangAmbigs.txt, kde x je kód jazyka ISO639-3. Tento soubor v kódovaní UTF-8 by měl obsahovat páry oddělené znakem „rovná se“ ve tvaru staráHodnota=nováHodnota.

Některé vestavěné nástroje poskytují funkce spojování (slučování) několika obrázků anebo souborů PDF do jednoho (pro pohodlnější rozpoznávání znaků), anebo dělení souborů PDF na menší, pokud jsou příliš veliké a způsobují „out-of-memory exceptions“.

DALŠÍ ZPRACOVÁNÍ

Chyby při rozpoznávání znaků se dají rozdělit do třech skupin. Časté jsou záměny velikosti písmen, např. „O“ — „o“, „Z“ — „z“, „S“ — „s“. Tyto chyby se dají opravit pomocí oblíbených textových editorů Unicode.

Další chyby jsou způsobeny procesem rozpoznávání - například chybějící diakritická znaménka, zaměněné znaky („1“ — „l“). Takové chyby je možné poměrně lehce opravit pomocí ověření pravopisu - kontroly překlepů (spellechecker). Vestavěné funkce pro další zpracování vám mohou pomoci s výše uvedeným chybami.

Poslední skupinou jsou chyby, které je těžké zjistit, protože jsoú to sémantické chyby, což znamená, že daná slova se nacházejí ve slovníku, ale jsou chybná z hlediska souvislosti napr. „cíp“ — „cip“ a pod. Tyto chyby si žádají, aby je opravil opravce ručně podle původního obrázku.

Zde jsou pokyny, jak opravit první dvě skupiny chyb vzniklých při rozpoznávání s pomocí zabudovaných funkcí:

Seskupení řádků. Řádky je potřeba seskupit tak, aby odpovídali odstavcům, ke kterým patří (1 odstavec = 1 řádek). Použijte na to funkci Odstranit zalomení řádků, která se nachází v nabídce Formát. Taková operace není potřebná pro básně.

Vyberte Změnit velikost písmen, z nabídky Formát, a potom První velké na opravu chyb velikosti písmen. Následně opravte ostatní chyby velikosti písmen (např. jména, názvy).

  1. Opravte překlepy pomocí Ověření pravopisu.
  2. Výše uvedené kroky by měly vyloučit většinu běžných chyb. Zůstávající sémantické chyby však musí opravit lidský opravce/vydavatel pozorným čtením a opravováním, aby byl výsledný dokument bez chyb.
  3. Pokud máte nějaké otázky, položte je ve fóru VietOCR.
